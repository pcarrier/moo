The local agent workbench.

Moo bundles a TypeScript harness, a Rust/V8 host, a Solid web shell, and an inspectable SQLite store into one hackable local system. It is packed with primitives for tools, memory, chat state, MCP, subagents, prompts, diffs, and previews.

Architecture

One binary. Local store.

Moo keeps the moving parts legible. The Rust process hosts V8, serves the web UI, stores state in SQLite, and exposes host operations. The TypeScript harness defines the agent-facing API. The Solid frontend gives you a live shell over the same local state.

Design rule: small primitives over a monolithic plugin platform. If something gets weird, you can inspect the object, ref, fact graph, chat trail, or file diff that produced it.

Run it

Install Moo.

Download a release binary and start the local server. If you are changing Moo itself, run process-compose from the repository.

macOS / Linux

curl -sf https://moo.pcarrier.com/install | sh
moo serve

Windows PowerShell

irm https://moo.pcarrier.com/install.ps1 | iex
moo serve

Sharp edges

Local tools, local caution.

Moo is for experimenting on your machine. Treat it like a developer harness, not a locked-down hosted assistant.

No sandbox by default

Tools can read/write files and run processes with the permissions of the process you start.

No approval gate

The harness favors direct experimentation. Add your own policy checks where you need them.

No hosted sync

State is local-first and SQLite-backed. Bring your own backup, sync, or remote model wiring.

Moving internals

The project is hackable and evolving; APIs may shift as the harness gets smaller and sharper.
